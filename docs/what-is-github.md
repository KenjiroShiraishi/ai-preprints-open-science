# 🐙 What is GitHub? / GitHubとは？

---

## 🌐 English

**GitHub** is an online platform for storing, sharing, revising, and collaborating on digital materials.

It is widely used by software developers, but it is also useful for researchers, educators, healthcare professionals, students, and independent scholars who want to make their work transparent, reusable, and openly available.

In the context of open science, GitHub can be used to share:

- Research notes
- Educational materials
- AI prompts and outputs
- Templates and checklists
- Data dictionaries
- Code and scripts
- Preprint-related documents
- Revision histories
- Public documentation

GitHub is not only a place to upload files.  
It is also a place to show **how knowledge develops over time**.

---

## 🧩 Key Concepts

### Repository

A **repository**, often called a “repo,” is a project folder on GitHub.

It can contain documents, code, images, PDFs, datasets, templates, and other materials related to a project.

For example, this project is stored in a GitHub repository so that others can read, reuse, cite, and improve the materials.

---

### README

A **README** file is the front page of a repository.

It usually explains:

- What the project is about
- Why it matters
- How to use the materials
- How to cite or reuse the work
- What license applies

A good README helps others understand the purpose and structure of the project.

---

### Commit

A **commit** is a saved change.

Each commit records what was changed and when.  
This makes it possible to trace the development of a project over time.

For open science, commits are useful because they provide a visible revision history.

---

### Fork

A **fork** is a personal copy of someone else’s repository.

Forking allows others to adapt, translate, expand, or reuse materials without directly changing the original repository.

This supports educational reuse and collaborative knowledge creation.

---

### Pull Request

A **pull request** is a proposed change to a repository.

If someone wants to improve a document, correct an error, or add a new resource, they can submit a pull request.  
The repository owner can then review and accept or reject the proposed change.

This allows collaboration while preserving responsibility and quality control.

---

### License

A **license** explains how others may use, share, adapt, or redistribute the materials.

For open science projects, clear licensing is important because it tells others what they are allowed to do with the work.

This repository uses the Creative Commons Attribution 4.0 International License (CC BY 4.0), which allows reuse and adaptation with proper credit.

---

## 🧠 Why GitHub Matters for Open Science

GitHub supports open science because it makes knowledge:

- **Accessible** — materials can be publicly shared
- **Reusable** — others can adapt and build on the work
- **Transparent** — revision histories can be preserved
- **Collaborative** — others can suggest improvements
- **Citable** — repositories can be linked to OSF, Zenodo, or DOIs
- **Educational** — students and practitioners can learn from real examples

In traditional academic publishing, only the final article is often visible.  
On GitHub, the process behind the work can also be made visible.

This is especially important when using AI in research and education.  
Prompts, outputs, revisions, and decisions can be documented so that others can understand how the final work was produced.

---

## ⚠️ Limitations

GitHub is useful, but it is not perfect.

Important limitations include:

- It may be difficult for beginners to use at first.
- It is not a substitute for peer review.
- Public repositories should not contain personal, confidential, or identifiable data.
- Medical or clinical materials must be anonymized carefully.
- Long-term preservation may require linking GitHub with OSF, Zenodo, or another repository.

GitHub should therefore be used as part of a broader open science workflow, not as the only place to preserve important research materials.

**GitHub**とは、デジタル資料を保存し、共有し、改訂し、共同で管理するためのオンラインプラットフォームです。

もともとはソフトウェア開発者によく使われてきましたが、現在では研究者、教育者、医療者、学生、独立研究者にとっても有用な道具になっています。

オープンサイエンスの文脈では、GitHubは次のような資料を公開・共有するために使うことができます。

- 研究メモ
- 教育資料
- AIプロンプトと出力
- テンプレートやチェックリスト
- データ辞書
- コードやスクリプト
- プレプリント関連資料
- 改訂履歴
- 公開用ドキュメント

GitHubは、単にファイルを置く場所ではありません。  
**知識がどのように作られ、修正され、発展していくかを見える形で残す場所**でもあります。

## 🧩 基本概念

### リポジトリ

**リポジトリ**とは、GitHub上のプロジェクト用フォルダのことです。  
英語では “repository” または “repo” と呼ばれます。

リポジトリには、文書、コード、画像、PDF、データ、テンプレートなど、プロジェクトに関係する資料をまとめて保存できます。

このプロジェクトも、他の人が読み、再利用し、引用し、改善できるようにGitHubリポジトリとして公開されています。

---

### README

**README**は、リポジトリの表紙にあたるファイルです。

通常、以下のような内容を書きます。

- そのプロジェクトが何か
- なぜ重要なのか
- どのように使うのか
- どのように引用・再利用できるのか
- どのライセンスが適用されるのか

よいREADMEは、他の人がプロジェクトの目的と構造を理解する助けになります。

---

### コミット

**コミット**とは、変更内容を保存することです。

どこを、いつ、どのように変更したのかが記録されます。  
これにより、プロジェクトがどのように発展してきたかを後から確認できます。

オープンサイエンスでは、コミット履歴は改訂過程の透明性を高めるために役立ちます。

---

### フォーク

**フォーク**とは、他の人のリポジトリを自分用にコピーすることです。

フォークすることで、元のリポジトリを直接変更せずに、翻訳、改訂、追加、教育利用などを行うことができます。

これは、教材や研究資料の再利用、改変、共同的な知識生産を支える仕組みです。

---

### プルリクエスト

**プルリクエスト**とは、リポジトリに対する修正提案のことです。

誤字を直す、新しい資料を追加する、説明を改善する、といった変更を提案できます。  
リポジトリの管理者は、その提案を確認し、採用するかどうかを判断します。

これにより、共同作業を可能にしながら、責任ある管理も維持できます。

---

### ライセンス

**ライセンス**とは、他の人がその資料をどのように使い、共有し、改変し、再配布できるかを示すルールです。

オープンサイエンスのプロジェクトでは、ライセンスを明確にしておくことが重要です。  
それにより、他の人が安心して資料を利用・改変・引用できます。

このリポジトリでは、Creative Commons Attribution 4.0 International License（CC BY 4.0）を用いています。  
これは、適切なクレジットを表示すれば、再利用や改変が可能なライセンスです。

---

## 🧠 なぜGitHubがオープンサイエンスに重要なのか

GitHubは、オープンサイエンスにおいて次のような役割を果たします。

- **アクセス可能性**：資料を公開できる
- **再利用可能性**：他の人が改変・応用できる
- **透明性**：改訂履歴を残せる
- **共同性**：他の人が改善提案できる
- **引用可能性**：OSF、Zenodo、DOIなどと接続できる
- **教育性**：実例を通じて学ぶことができる

従来の学術出版では、最終的な論文だけが見えることが多くあります。  
しかしGitHubでは、その成果物が作られる過程も見える形で残すことができます。

これは、AIを研究や教育に使う場合に特に重要です。  
プロンプト、出力、修正、判断過程を記録することで、最終成果物がどのように作られたのかを他の人が確認しやすくなります。

---

## ⚠️ 限界

GitHubは便利ですが、万能ではありません。

主な限界は以下の通りです。

- 初心者には最初やや分かりにくい
- 査読の代わりにはならない
- 個人情報、機密情報、識別可能なデータを公開してはいけない
- 医療・臨床関連資料は慎重に匿名化する必要がある
- 長期保存のためには、OSF、Zenodo、その他のリポジトリと組み合わせることが望ましい

そのため、GitHubは重要な研究資料を保存する唯一の場所ではなく、より広いオープンサイエンスのワークフローの一部として使うべきです。

このプロジェクトでは、GitHubを以下の目的で使用しています。

- 入門的な教育資料を共有する
